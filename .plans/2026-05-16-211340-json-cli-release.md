# Goal

Release a `slopless` npm binary that always emits JSON.

Users should be able to install only `slopless` and run `npx slopless` without creating `.textlintrc.json` or installing `textlint` separately.

# Approach

1. Add `src/cli.ts`.
   - Call textlint's CLI implementation.
   - Inject `--preset slopless`.
   - Inject `--rules-base-directory` pointing at the package's parent `node_modules`.
   - Inject `--format json`.
   - Use `**/*.md` when the user passes no files.
   - Pass through all other textlint options.
   - Reject user-supplied `--format` / `-f` because the command is JSON-only.
2. Update package metadata.
   - Bump version to `0.2.0`.
   - Add `bin.slopless`.
   - Move `textlint` to runtime dependencies so the wrapper works alone.
3. Update README.
   - Show `npm install -D slopless`.
   - Show `npx slopless`.
   - State that output is always JSON.
4. Verify.
   - `npm run validate`.
   - Pack and install tarball into a fresh temp project.
   - Run `npx slopless sample.md` and parse JSON.
   - Publish `slopless@0.2.0`.
   - Install `slopless@0.2.0` from npm in a fresh temp project and parse JSON.

# Key Decisions

- JSON-only means the wrapper owns output format and does not expose textlint's formatter choice.
- The wrapper still delegates linting and reporting to textlint.
- No `init` command in this release because the zero-config command is the simpler user path.
